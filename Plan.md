# Техническое задание: Revit Plugin - "Smart Group Manager"

## 1. Обзор проекта
Создание плагина для Autodesk Revit, который позволяет временно разгруппировывать элементы с сохранением метаданных (имя, положение, состав) и последующим восстановлением группы в один клик.

## 2. Стек технологий
* **Язык:** C# (.NET 4.8 / .NET Core в зависимости от версии Revit)
* **API:** Revit API
* **UI:** WPF (Windows Presentation Foundation) для панели менеджера
* **Хранение данных:** Shared Parameters + Extensible Storage

---

## 3. Этапы разработки

### Этап 1: Подготовка инфраструктуры данных
**Задача:** Создать механизм идентификации элементов, вышедших из группы.
* [ ] **Кнопка "Настройка проекта":** Автоматическое создание общего параметра `Original_Group_GUID`.
    * *Тип:* Text (Текст).
    * *Категории:* Все категории модели (Walls, Doors, Windows, Furniture и т.д.).
    * *Свойства:* Видимый, но желательно ReadOnly (только для чтения) через API.
* [ ] **Extensible Storage (Registry):** Создание схемы данных внутри файла Revit для хранения списка "разобранных" групп (GUID, Имя, Координаты XYZ, Угол поворота).

### Этап 2: Логика разгруппировки (Unwrap)
**Задача:** Корректно "пометить" элементы перед удалением группы.
* [ ] Получение текущего `GroupInstance`.
* [ ] Генерация уникального `System.Guid`.
* [ ] Цикл по `Group.GetMemberIds()`: запись GUID в параметр `Original_Group_GUID` каждого элемента.
* [ ] Сохранение данных в реестр проекта:
    * Исходное имя типа группы.
    * Точка вставки (`Location.Point`).
    * Вектор поворота.
* [ ] Вызов метода `doc.Delete(groupId)`.

### Этап 3: Панель управления (Manager UI)
**Задача:** Визуальный интерфейс для пользователя (Dockable Pane).
* [ ] **Таблица со списком групп:**
    * Столбец "Имя": Возможность редактирования (Rename).
    * Столбец "Статус": Индикатор (Собрана / Разобрана).
    * Столбец "Действие": Кнопка "Собрать" напротив каждой строки.
* [ ] **Синхронизация:** При переименовании в таблице — обновлять данные в реестре проекта.

### Этап 4: Логика сборки (Reassemble)
**Задача:** Найти помеченные элементы и объединить их.
* [ ] Фильтрация элементов в модели по значению `Original_Group_GUID == Selected_GUID`.
* [ ] Проверка состава: если часть элементов удалена, собирать из того, что осталось.
* [ ] Создание новой группы через `doc.Create.NewGroup(elementIds)`.
* [ ] Присвоение группе имени из менеджера (если тип с таким именем уже есть — использовать его, если нет — переименовать новый тип).
* [ ] Очистка параметра `Original_Group_GUID` у элементов после успешной сборки.

---

## 4. Обработка исключительных ситуаций
1. **Удаление хоста:** Если стена (хост) была удалена, пока группа была разобрана, окно/дверь внутри группы могут быть удалены Revit-ом автоматически. Плагин должен игнорировать отсутствующие ID.
2. **Переименование:** Если пользователь вручную создал группу с таким же именем, плагин должен добавить суффикс (например, `_1`) во избежание конфликта имен типов.
3. **Закрытие файла:** Все данные должны сохраняться внутри `.rvt` файла, чтобы при открытии на другом ПК список групп в менеджере был актуален.
